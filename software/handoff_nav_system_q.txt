C) SPACEMOUSE CARDINAL DOUBLE-TILT MACROS (_poll_spacemouse)
------------------------------------------------------------
Preprocessor: _preprocess_directional_double_taps

When nav_scope is console OR knobs OR faders (NOT editor nor transport):
  - Same XY cardinal tilt twice within DOUBLE_TAP_S (~0.32s) fires a shortcut
    and consumes the second directional (no extra single-ring nav for it).

CURRENT mappings:
  - DOUBLE RIGHT → _open_stage_editor(ch, selected_stage_key) for focused strip
    channel index (console selected_channel vs knob_focus_channel vs fader_focus_channel).

  - DOUBLE LEFT → nav_scope console, console_row stages, preserve channel focus;
    clears _transport_entered_from; normalize + redraw.

  - DOUBLE DOWN → _focus_transport_play_cell(source="spacemouse_double_down")
    with editor_channel set to focused strip index; _sync_from_engine.

REMOVED by user request:
  - DOUBLE UP → used to _enter_fader_row; removed as causing problems.
  - UP is explicitly bypassed in double-tap logic; each UP is normal nav only.
  - On every UP, double-tap latch state is cleared so vertical motion does not
    prime accidental RIGHT/LEFT/DOWN doubles.

Shared helper: _focus_transport_play_cell — also used from
_enter_editor_focus_transport_play (source still editor_scope_toggle).
