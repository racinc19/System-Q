WHAT THE FEATURE IS SUPPOSED TO DO
----------------------------------
A) ~3 s sustained tilt on the SpaceMouse Y axis toward “down” (pygame positive
   Y at direction_threshold) emits one synthetic event: down_hold from
   pol_visualizer.SpaceMouseController.poll().

B) system_q_console._poll_spacemouse() strips down_hold and calls
   _handle_down_hold_jump(), which:
   - nav_scope == "console" (rows stages/record/footer/knob/fader)
     → _open_stage_editor(selected_channel, selected_stage_key)
   - nav_scope == "editor" → _enter_fader_row(editor_channel, …)
   - nav_scope == "faders"  → _exit_faders_to_console()


KEY IMPLEMENTATION FILES
------------------------
  software/pol_visualizer.py
    - xy_z_max_for_nav (default 0.38): |Z| below this allows XY + down-hold
      timer; previously 0.22 reset the 3 s timer under light cap pressure.
    - Down-hold uses raw y_val >= direction_threshold, not only cardinal
      dominance, so slight diagonal tilt still counts.
    - Short release without 3 s emits one "down" for normal stepping.

  software/system_q_console.py
    - _handle_down_hold_jump(), twist enter/exit unchanged elsewhere.
    - SYSTEM_Q_NAV_DEBUG=1 prints when down_hold is emitted (visualizer) and
      when jump runs (console).


SILENT NO-OPS (COMMON GOTCHAS)
------------------------------
1) SYSTEM_Q_AGENT_PROOF set in the environment
   → _agent_proof_engages_editor_cells() is True
   → _handle_down_hold_jump returns immediately (no editor).
   Launch without proof batches / clear SYSTEM_Q_AGENT_PROOF.

2) Gesture is TILT Y “down”, not necessarily pushing the cap (Z). Heavy Z
   can still block XY if |z| >= xy_z_max_for_nav.

3) SpaceController not opened / wrong device index → poll returns empty; use
   SYSTEM_Q_SPACEMOUSE_INDEX if multiple joysticks.
